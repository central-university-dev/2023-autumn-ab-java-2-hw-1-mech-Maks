HashMap - представляет реализацию хет-таблицы в Java

0) HashMap обеспечивает доступ по ключу в среднем за O(1) (при равномерном распределении)
1) HashMap использует цепное хеширование, то есть каждый бакет представляет из себя список элементов с таким же хэшем
2) При большом количестве элементов в бакете его структура меняется на бинарное дерево (при нормально распределении
такое не должно случаться)
3) HashMap не является часть java collections framework, так как представляет из себя не набор, а пары ключ-значение
4) Так как HashMap не является частью java collections нельзя использовать интерфейс Iterable, но Iterator можно
5) При создании HashMap можно задать емкость и коэффициент заполненности, при достижении которого размер HashMap
увеличится вдвое
6) HashMap может иметь null как в качестве ключа, так и в качестве значения
7) HashMap является несинхронизированным, то есть потокобезопасным. Для безопасности лучше использовать ConcurrentHashMap
